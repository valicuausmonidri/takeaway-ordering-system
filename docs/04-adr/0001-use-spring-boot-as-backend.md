# 1. 选择Spring Boot作为后端框架
*   日期: 2025-01-04
*   决策者: 黄鸿鑫

## 上下文与问题陈述
外卖订餐系统需要一款Java后端框架，用于快速开发API接口、整合MySQL数据库、实现订单/菜品/用户的核心业务逻辑，要求框架成熟稳定、学习成本低、适合中小型项目快速落地。

## 决策因素
*   因素 1：开发效率 - 能否减少配置代码，快速搭建项目骨架
*   因素 2：生态适配 - 能否无缝整合MyBatis（数据库交互）、Spring Security（基础权限）
*   因素 3：学习成本 - 团队（个人）能否快速上手，文档是否丰富

## 考虑的方案
*   方案 A：Spring Boot 2.7.x - 约定优于配置，自动配置核心组件，社区文档丰富
*   方案 B：Spring MVC + XML配置 - 需手动编写大量配置，开发效率低，已逐步淘汰

## 决策结果
选择[方案 A：Spring Boot 2.7.x]，因为它：
1. 自动配置Tomcat、DataSource等核心组件，无需XML配置，10分钟即可搭建基础项目；
2. 生态完善，可直接集成MyBatis-Plus简化CRUD操作，适配项目需求；
3. 官方文档详细，国内社区资源丰富，遇到问题可快速找到解决方案，降低学习成本。

## 后果
### 积极后果
- 后端开发效率提升50%，专注业务逻辑而非配置；
- 后续扩展功能（如添加Redis缓存、消息队列）时，有成熟的Starter依赖支持。

### 消极后果
- 自动配置逻辑较黑盒，初期排查配置问题需熟悉Spring Boot自动配置原理；
- 对简单项目而言，存在轻微“过度封装”，但影响可忽略。
